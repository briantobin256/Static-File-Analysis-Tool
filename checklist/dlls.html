Description:

Dynamic Link Libraries are libraries of code that contain common functions. Different DLL's specialise in different types of functionality. Each imported DLL will have a list of imported functions.


Imported DLL's

The imported DLL's can give a hint of some general things that may be happening.
Common DLL's to look out for:
KERNEL32 is the most common DLL that provides core Windows functionality.
ADVAPI32 gives access to more advanced Windows functionality such as accessing the registry.
NTDLL is used to interface directly with the Windows kernel. Provides advanced functionality not normally available to programs such as manipulating running processes.
USER32 contains functions for interacting with user-interface components.
WININET contains functions for high-level networking such as HTTP or FTP protocols.


Imported Functions

The imported functions can be very useful for finding out specific actions the file will perform. The function names are usually very self explanatory but a quick search on the internet can give even more information if needed.


Your Task:

To look through the imported functions and note the ones that you find interesting. Come up with a hypthesis of what the file might do when executed based on these imported functions.


Note:

The Import Directory Table contains relative pointers to the location of the imported DLL's. The Import Address Table contains relative pointers to the location of the imported functions.
